Балабанова Елизавета. Твоя моя понимай. Интернет ищет язык межпрограммного общения
Переход на более совершенные версии ПО часто сдерживается необходимостью быстро конвертировать «старые» данные клиента в новый формат. Среди средств выполнения такой процедуры язык XML становится все более популярным.
Главная задача XML (etensible markup language -- расширенный язык разметки) -- это организация взаимодействия между программами. Однако сейчас у XML появился шанс стать универсальным языком представления данных для всего Интернета. Информационное агентство NewsFactor цитирует аналитика компании Forrester Галена Шрека, который приводит такие новые стандарты, как Simple object access protocol (SOAP) и Web services description language (WSDL), в качестве «высшей формы XML-интеллекта». «У нас есть XML -- сейчас нам нужен интеллект для работы с ним», -- заявил аналитик.
Наша страна не стоит в стороне от «языкового движения». Например, рядом компаний был разработан стандарт в формате XML, названный CommerceML. Среди участников консорциума -- Microsoft, Intel, 1С, Extra.Ru, Port.ru, Price.ru. CommerceML был создан для унификации обмена коммерческой информацией между различными организациями. В фирме 1С «Известиям» так пояснили выбор языка описания деловой информации: «Сфера применения XML не ограничена только интернет-решениями, он успешно используется и при разработке систем конвертации данных, стандартов и форматов обмена на уровне предприятий и отраслей. Поэтому можно ожидать, что количество основанных на XML языков, схем и различных их диалектов будет и дальше расти».
Любому искусственному языку всегда угрожает «эффект Эсперанто» -- на этом языке можно даже стихи писать, но вытеснить «живые» языки межнационального общения он не смог. Судя по всему, XML это не грозит, поскольку в своей области он умудрился стать «престижным» языком. В московском представительстве Oracle «Известиям» рассказали, что эти три буквы вызывают даже у неискушенных клиентов благоговение. Многие российские предприятия в свое время сами написали километры ПО. Теперь для солидности или для соответствия международным стандартам предприятия начинают внедрять «фирменный» софт и проблема переноса данных из старой программы в новую становится для них критической. Как только клиент слышит, что обмен можно организовать с помощью XML, он сразу же успокаивается.
